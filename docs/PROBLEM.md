# Problem Statement

## Contexto

Muchas personas tienen dificultades para llevar un control claro y constante de sus gastos personales. 
Aunque existen herramientas como hojas de cálculo o aplicaciones móviles, estas suelen requerir 
configuración previa, tienen funcionalidades innecesarias o no explican claramente cómo se procesan los datos.

Como resultado, los usuarios terminan abandonando estas herramientas y vuelven a perder visibilidad 
sobre sus finanzas personales.

## Usuario objetivo

Personas que desean:
- Entender en qué gastan su dinero
- Llevar un control simple y transparente
- No depender de aplicaciones cerradas o complejas

No se requiere conocimiento técnico previo.

## Problema principal

- Falta de visibilidad sobre los gastos mensuales
- Herramientas existentes poco flexibles o demasiado complejas
- Procesos manuales propensos a errores

## Objetivo del proyecto

Desarrollar una aplicación en Python que permita registrar, almacenar y consultar gastos personales 
de forma sencilla, clara y estructurada, priorizando la transparencia del procesamiento de los datos.

## Objetivo técnico

Este proyecto tiene como objetivo demostrar:
- Diseño de una aplicación modular
- Separación de responsabilidades
- Buenas prácticas de control de versiones con Git
- Pensamiento orientado a producto y mantenibilidad

No es objetivo de este proyecto crear una aplicación comercial completa ni una interfaz gráfica avanzada.
